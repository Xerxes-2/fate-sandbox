# Showrunner audit on an engine-owned sync spawn seam; retire the runtime subagent-framework dependency

The `timeline-showrunner` auditor was the last runtime consumer of `pi-subagents`: a `.pi/agents/` markdown agent invoked by the GM model through the framework's `subagent` tool, with a `tool_call` hook (`task-injection.ts`) rewriting the framework's SINGLE/PARALLEL/CHAIN input shapes to append `<timeline_state_context>`. The decision: **move the audit onto an engine-owned synchronous spawn seam (`run_showrunner_audit`), and hard-cut `pi-subagents` from the project packages in the same migration.** The GM model still decides WHEN to audit (tool-policy triggers unchanged — this is deliberately _not_ the engine-triggered variant); the engine owns everything else: a typed `TimelineShowrunnerInput` tool schema normalized at the tool boundary, prompt assembly (persona + input + `<timeline_state_context>` embedded directly, no input-shape guessing), a blocking `pi -p` child via `node:child_process` (persona passed as `--system-prompt` text to preserve the old `systemPromptMode: replace` semantics; child loads only `extensions/subagents/timeline/index.ts` for `lookup` via `--no-extensions -e … --no-builtin-tools --no-context-files --no-skills`, sessions under gitignored `.pi/agent/showrunner-sessions`, no `--model` pin per the ADR 0005 amendment rationale), and a TypeBox `TimelineShowrunnerOutput` return-trip gate. Failures (spawn, 5-minute timeout, schema rejection) return a structured failure branch — schema errors list the offending fields, never leak raw child text, and the engine performs **zero retries**; the GM decides whether to re-call.

This applies ADR 0005 to a case where the framework had no capability gap, so the reason for the migration is recorded separately. `pi-subagents` satisfied the read-only and independence requirements. The migration instead reduces runtime surface and maintenance: the showrunner path exercised ~10% of a 38K-line runtime dependency shipped to players; the only actively maintained glue (`task-injection.ts` + its tests) existed to chase the framework's tool-input schema across versions, failing silently into a degraded-context path; and the one invariant that matters most — executable-JSON-only output — was a prompt promise with no engine gate, which the charter ("Prompt 不是防线") says must be a schema gate we would have to build anyway. Migrating deletes the glue and removes the unsupported `subagent` tool entry from the GM model (charter hard-cut: a reachable old entrance keeps teaching the model to use it), and adds the missing output gate in the same motion. The audit's context block is now named in the glossary as the **Showrunner Projection** — a _semi-privileged_ read model (offscreen event digests, actor agenda/knowledge-lens facets; never raw secret slots) — resolving the prior mislabeling of the timeline context as a Public Game State Projection consumer.

Rejected alternatives: **keep pi-subagents (A)** — harmless short-term but every upstream release is a task-injection regression risk and the player install carries a framework for one blocking call; **inline the rubric into the settlement prompt (C)** — 141 lines of per-turn prompt budget for an occasional check, and an auditor's value is precisely that it has no path dependency on the GM's own generation; **engine-triggered audits (B2)** — changes the settlement flow and "drift" has no mechanical trigger predicate yet; it remains a small increment on this seam if ever wanted; **deterministic ledger reconciliation (D)** — orthogonal and still desirable: the mechanical audit steps (hook counting, no-cost streaks, UTC format) should keep sinking into engine invariants, shrinking what the LLM auditor reads. **B-async** (detached + harvest, matching the backstage path) remains an option if blocking latency becomes unacceptable. The verdict applies to the next turn, so the domain can tolerate delayed retrieval. Acceptance invariants for any future substrate swap: child has `lookup` only (no write tools, no `read`/`bash`); verdicts reach the GM only through the output schema gate; the child loads no project extension, AGENTS.md, or skills; the Showrunner Projection keeps filtering `actorSecrets`/`secretEventLog`/`campaignSecrets`; child sessions stay in a gitignored directory; and spawn/timeout failures are loud, never a silent pass.
